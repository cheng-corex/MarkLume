## 报告安全漏洞

MarkLume 是一个本地 Markdown 阅读器，安全性是我们的首要关注。

如果你发现了安全漏洞，**请不要公开披露**，而是通过以下方式私下报告：

1. 在 [GitHub Issues](https://github.com/cheng-corex/MarkLume/issues) 中创建 Issue
2. 在标题中添加 `[安全]` 前缀
3. 详细描述漏洞类型、影响范围和复现步骤

我们会在收到报告后尽快评估和处理，通常会在 **7 天内**回复。

## 安全设计原则

- **本地优先**: 所有文件操作均在本地完成，不涉及网络请求
- **HTML 安全过滤**: 所有 Markdown 渲染内容经过 DOMPurify 清洗，防止 XSS 攻击
- **文件访问限制**: Rust 后端对文件读取有扩展名和大小限制
- **无远程调用**: 应用不包含任何遥测或远程执行代码
