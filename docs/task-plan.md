# MarkLume 任务计划

## 阶段划分总览

| 阶段 | 内容 | 状态 |
|------|------|------|
| 第一阶段 MVP | 基础阅读器核心功能 | ✅ 当前阶段 |
| 第二阶段 | 阅读体验增强 | ⏳ 待开始 |
| 第三阶段 | 文件夹阅读模式 | ⏳ 待开始 |
| 第四阶段 | 搜索功能 | ⏳ 待开始 |

---

## 第一阶段 MVP 任务分解

### 第 1 步：初始化项目

**任务清单：**
- [ ] 创建 Tauri 2 + Vite + React + TypeScript 项目
- [ ] 确认 `npm run dev` 可以启动
- [ ] 确认 Tauri 桌面窗口可以打开
- [ ] 建立基础目录结构
- [ ] 创建 docs 目录和开发文档

**验收标准：**
1. 项目能启动
2. Windows 桌面窗口正常显示
3. 前端热更新正常

---

### 第 2 步：基础布局

**任务清单：**
- [ ] 顶部工具栏
- [ ] 左侧侧边栏
- [ ] 中间阅读区
- [ ] 右侧大纲区预留
- [ ] 基础样式
- [ ] 空状态页面

**验收标准：**
1. 界面结构稳定
2. 窗口缩放正常
3. 没有明显布局错乱

---

### 第 3 步：打开文件

**任务清单：**
- [ ] 点击"打开文件"按钮
- [ ] 弹出系统文件选择框
- [ ] 只允许选择 `.md`、`.markdown`、`.txt`
- [ ] 使用 Tauri 后端读取文件
- [ ] 前端显示文件名和原始内容

**验收标准：**
1. 能打开普通 Markdown 文件
2. 能处理文件不存在、权限不足、格式不支持
3. 错误提示清晰

---

### 第 4 步：Markdown 渲染

**任务清单：**
- [ ] 安装并配置 markdown-it
- [ ] 支持常见 Markdown 语法
- [ ] 安装并配置 DOMPurify
- [ ] 渲染 HTML 前必须清洗
- [ ] 增加 markdown.css，使内容适合阅读

**验收标准：**
1. 标题、列表、引用、表格、代码块显示正常
2. 恶意 HTML 不会直接执行
3. 阅读区样式美观、清晰

---

### 第 5 步：主题和字号

**任务清单：**
- [ ] 亮色主题
- [ ] 暗色主题
- [ ] 跟随系统主题
- [ ] 字号调整
- [ ] 行高调整
- [ ] 内容宽度调整

**验收标准：**
1. 切换主题后界面即时更新
2. 重启后配置保留
3. Markdown 阅读区样式不崩

---

### 第 6 步：最近文件和本地配置

**任务清单：**
- [ ] 打开文件后加入最近文件
- [ ] 最近文件去重
- [ ] 最近文件限制数量（最多 20 个）
- [ ] 保存配置到本地
- [ ] 启动时恢复配置
- [ ] 启动时可自动显示上次文件

**验收标准：**
1. 重启后最近文件还在
2. 点击最近文件可以重新打开
3. 文件不存在时要提示并允许从列表移除

---

### 第 7 步：大纲功能

**任务清单：**
- [ ] 提取 Markdown 标题
- [ ] 生成标题 ID
- [ ] 渲染右侧大纲
- [ ] 点击大纲跳转到对应标题
- [ ] 滚动时高亮当前标题

**验收标准：**
1. H1-H6 层级正确
2. 重名标题 ID 不冲突
3. 跳转位置准确

---

### 第 8 步：文件夹阅读模式

**任务清单：**
- [ ] 打开文件夹
- [ ] 扫描目录内 Markdown 文件
- [ ] 左侧显示文件树
- [ ] 点击文件打开
- [ ] 支持上一个/下一个文件
- [ ] 记住上次目录

**验收标准：**
1. 文件树显示正常
2. 嵌套目录显示正常
3. 只展示支持的文件类型
4. 大目录不要明显卡死

---

### 第 9 步：搜索功能

**任务清单：**
- [ ] 当前文件搜索（输入关键词、高亮匹配、上/下一个、匹配数量）
- [ ] 文件夹搜索（遍历目录、显示文件名/片段/位置、点击结果打开文件）

**验收标准：**
1. 搜索结果准确
2. 中文搜索正常
3. 大小写处理合理
4. 搜索无结果时提示清晰

---

### 第 10 步：打包安装

**任务清单：**
- [ ] 配置应用名称
- [ ] 配置应用图标
- [ ] 配置版本号
- [ ] 配置 Windows 打包
- [ ] 生成安装包
- [ ] 验证安装、启动、卸载

**验收标准：**
1. 可以生成 Windows 安装包
2. 安装后可以从开始菜单启动
3. 桌面快捷方式正常
4. 卸载正常
5. 安装后的应用能正常打开 Markdown 文件
